# 🧭 graph-pilot

> Автопилот над [Graphify](https://github.com/safishamsi/graphify). Ты говоришь, что хочешь
> сделать с проектом — на русском, человеческим языком. Скилл сам подбирает правильную
> команду Graphify, запускает её и **бережёт твои токены**.
>
> Лид-магнит к ролику «Graphify» (Никита Велс | AI).

У Graphify **больше сорока команд и флагов**: `query`, `path`, `explain`, `--mode deep`,
`--cluster-only --resolution`, `--exclude-hubs`, `extract --postgres`, `prs --triage`,
обсидиан, вики, MCP, neo4j... Запомнить это невозможно, и легко случайно запустить дорогой
режим там, где хватило бы бесплатного. Этот скилл — переводчик «намерение → команда» с
вшитыми токен-гардами.

---

## Когда срабатывать

Активируй скилл, когда пользователь по-человечески формулирует намерение про код, а не
команду. Примеры триггеров:
- «разберись, как тут устроена авторизация / оплата / вебсокеты»
- «как связаны вот этот файл и вот тот»
- «объясни, что делает вот этот класс»
- «я поменял кучу файлов, обнови граф»
- «почему так много токенов уходит на Graphify»
- «стоит ли вообще строить тут граф»
- «собери мне вики / обсидиан по этому проекту»

## 🧠 ГЛАВНАЯ ТАБЛИЦА: НАМЕРЕНИЕ → КОМАНДА

Сопоставь фразу пользователя с намерением и выполни **ровно одну** команду. Не запускай
лишнего. Если граф ещё не построен (`graphify-out/graph.json` нет) — сначала Шаг А.

| Что хочет пользователь (по-русски) | Команда Graphify | Стоит токенов? |
|---|---|---|
| Понять, как устроена фича / подсистема | `graphify query "<вопрос>"` | дёшево (читает граф, не весь код) |
| Найти связь между двумя файлами/сущностями | `graphify path "<A>" "<B>"` | дёшево |
| Объяснить один класс/функцию | `graphify explain "<сущность>"` | дёшево |
| Поменял файлы, нужно освежить граф | `graphify . --update` | **0 токенов** (AST) |
| Построить граф впервые | `graphify .` | **0 токенов** (AST, код локально) |
| Собрать вики по проекту | `graphify . --wiki` | дёшево |
| Собрать обсидиан-базу | `graphify . --obsidian` | дёшево |
| Кластеры слишком крупные/мелкие | `graphify . --cluster-only --resolution 1.5` | **0 токенов** |
| Утилиты-хабы засоряют граф | `graphify . --cluster-only --exclude-hubs 99` | **0 токенов** |
| Граф «грязный» после рефакторинга | `graphify extract . --force` | **0 токенов** (AST) |
| Глубокий семантический разбор документов / PDF | `graphify . --mode deep` | ⚠️ **ПЛАТНО** — см. гард ниже |

## 🛡 ТРИ ГАРДА (выполнять ВСЕГДА, до запуска команды)

### Гард 1 — Стоит ли вообще строить граф

Если граф ещё не построен, прикинь размер кодовой базы перед `graphify .`:

```bash
find . -type f \( -name '*.py' -o -name '*.js' -o -name '*.ts' -o -name '*.go' \
  -o -name '*.rs' -o -name '*.java' -o -name '*.rb' -o -name '*.cpp' -o -name '*.c' \) \
  -not -path '*/node_modules/*' -not -path '*/.venv/*' -not -path '*/dist/*' \
  -not -path '*/build/*' | wc -l
```

- **< 100 файлов** → скажи: «Граф тут почти не сэкономит — обычный поиск и так быстрый. Строить?»
  и дождись подтверждения. Не строй молча.
- **100–500** → строй (`graphify .`), экономия будет умеренная.
- **> 500** → строй, тут граф реально окупается.

### Гард 2 — `--mode deep` только осознанно

`--mode deep` гонит документы через языковую модель = **твои токены**. Дефолтный
`graphify .` разбирает код локально tree-sitter'ом = **0 токенов**.

Запускай `--mode deep` **только если** одновременно:
1. В проекте реально много документации / PDF / диаграмм, И
2. Пользователь явно попросил «глубокий разбор смысла документов».

Иначе — используй обычный `graphify .` и скажи: «Хватит бесплатного AST-режима,
`--mode deep` тут только сожжёт токены».

### Гард 3 — Always-on хук НЕ ставить вслепую

Граф полностью рабочий через явные команды (`query`, `explain`, `path`). Команда
`graphify claude install` вшивает PreToolUse-хук в Claude Code, и **на части свежих
сборок этот хук слетает и блокирует tool-вызовы** — Claude перестаёт читать файлы.

Поэтому:
- **Не запускай `graphify claude install` по своей инициативе.**
- Если пользователь просит always-on — сначала `claude --version`, потом спроси явно:
  «Хук может слететь на свежих сборках Claude Code и заблокировать чтение файлов.
  Ставить? Если что — откат в Шаге Отката ниже».
- По умолчанию рекомендуй **post-commit хук без PreToolUse** — он только перестраивает
  граф после коммита и **ничего не ломает**: `graphify hook install` (0 токенов, AST).

## 📋 АЛГОРИТМ РАБОТЫ

### Шаг А — Убедиться, что Graphify установлен и граф есть

```bash
graphify --version    # если command not found → установить (Шаг Б)
```

Если `graphify-out/graph.json` нет — построй граф (после Гарда 1):

```bash
graphify .            # AST, 0 токенов. В PowerShell — без ведущего слэша
```

### Шаг Б — Установка (если Graphify нет)

```bash
uv tool install graphifyy   # двойная y! пакет на PyPI — graphifyy, команда — graphify
graphify install            # зарегистрировать скилл Graphify в проекте
```

Если нет `uv` → `pipx install graphifyy`. **Не предлагай `pip install`** (PATH).

### Шаг В — Подобрать команду по таблице «намерение → команда» и выполнить

Возьми фразу пользователя → найди строку в главной таблице → выполни **одну** команду.
Проговори вслух, что и почему запускаешь.

### Шаг Г — Если что-то пошло не так

| Симптом | Что делать |
|---|---|
| `graphify: command not found` | `uv tool install graphifyy` (не `pip`) |
| Поставил `graphify` (одна `y`) | Пакет — `graphifyy`, с двумя `y` |
| `/graphify .` ругается в PowerShell | Пиши `graphify .` без слэша |
| Токены улетели | Запустили `--mode deep` зря → вернись к `graphify .` (AST) |
| После `graphify claude install` Claude не читает файлы | **Шаг Отката** ↓ |

### Шаг Отката — если хук сломал Claude Code

1. Открой `~/.claude/settings.json` (или `.claude/settings.json` в проекте).
2. Удали блок `hooks` с упоминанием `graphify` (секция `PreToolUse`).
3. Перезапусти Claude Code. Граф на месте — работай через `query`/`explain`/`path`.

## ✅ Что этот скилл гарантирует

- Тебе **не надо помнить 40 команд** — говоришь намерение, скилл подбирает команду.
- Не запустит дорогой `--mode deep` там, где хватает бесплатного AST.
- Не построит граф на крошечном проекте без твоего согласия.
- **Не вшьёт ломающий хук вслепую** — твой Claude Code в безопасности.
